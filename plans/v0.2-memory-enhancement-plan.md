## §1 데이터 모델 (리뷰 반영 — FTS5 제거)

```sql
CREATE TABLE memory (
  id            TEXT PRIMARY KEY,        -- UUID
  text          TEXT NOT NULL,
  embedding     BLOB,                    -- float32[1024] (NULL=임베딩 대기 → LIKE 폴백 대상)
  type          TEXT NOT NULL,           -- 선호|사실|일정|관계
  importance    INTEGER NOT NULL,        -- 1~10
  created_at    TEXT NOT NULL,
  last_accessed TEXT,                    -- 세션당 1회 갱신(M-D14)
  valid_to      TEXT,                    -- 일정형 만료일만 사용(M-D12). NULL=무기한
  source        TEXT,
  verified      INTEGER NOT NULL DEFAULT 0
);

CREATE TABLE pending_extraction (        -- M-D5 2단 분리: 빠른 기록 → 유휴 시 멱등 처리
  id            TEXT PRIMARY KEY,
  transcript    TEXT NOT NULL,           -- 사용자 턴 중심 원시 대화
  created_at    TEXT NOT NULL,
  attempts      INTEGER NOT NULL DEFAULT 0
);

-- core 프로필(①): 구조화 JSON 1건(이름·직업·관심사·말투 선호·금기) — AppSettingsStore 확장
-- 세션 요약(③): 직전 세션 요약 1건 — 별도 저장, 브리핑이 재사용(P2-11: BriefingCache 와 분리)
```

- 키워드 폴백 검색: `SELECT ... WHERE text LIKE '%' || ? || '%'` (인덱스 불요·수백 건 스캔).
- 임베딩 차원 가드: 저장 차원 ≠ 질의 차원이면 해당 레코드는 코사인 제외(LIKE 폴백) + 재임베딩 대상 표시.

## §3 쓰기 경로 (2단 분리 — P0-1 반영)

```
[1단: 빠른 기록 — 사용자 대기 0]
 대화 경계(newChat 등) → 사용자 턴 중심 transcript를 pending_extraction 에 동기 INSERT

[2단: 유휴 처리 — 멱등]
 트리거: 앱 시작 / 포그라운드 복귀 / newChat 직후 (모두 포그라운드)
 for row in pending_extraction:
   LLM 추출(M0 검증 프롬프트 + 부정문 few-shot, JSON 실패→attempts+1, 3회 초과 폐기)
   → 항목별 임베딩(8081; 실패 시 embedding=NULL 저장)
   → 룰 NOOP(θ_same=0.75) 통과분만 INSERT(verified=0)
   → row DELETE (전 과정 완료 시에만 — 중간 사망 시 다음 기회 재처리)
```

## §4 읽기 경로

- `send()` 직전: 입력 임베딩(실패 시 LIKE) → 유효 기억(일정형 만료 제외) 점수화 → 후보 top-N → **동주제 dedup(M-D9', 최신 우선)** → 3~5건 system 주입(M-D10 순서) → last_accessed 갱신(세션당 1회).
- 브리핑: 동일 검색 + 세션 요약(③) 주입.
- 명시 저장: `save_memory` tool(좁은 description) → 즉시 INSERT(verified=1 — 사용자 명시이므로 검토 불요).

## §7 테스트 계획 (리뷰 P1-7 보강 — "엄중 검증" 5대 시나리오 + 결함 항목)

**5대 E2E 시나리오** (시뮬레이터 + DB 직접 검사):
1. **선호 기억 회상**: "나 매운 음식 못 먹어" → 대화 경계 → DB 저장 확인 → 새 대화 "저녁 메뉴 추천" → 주입·응답 반영
2. **일정 기억 + 수명**: "6월 말 부산 출장" 저장 → 회상 확인 / 만료 일정(`valid_to` 과거) 미주입 확인
3. **모순 발화 처리**: "요즘 매운 것 먹게 됐어" → append-only ADD + **읽기 시 최신 우선 dedup**(M-D9') → 응답이 최신 기준
4. **명시 저장 tool**: "내 생일 10월 3일인 거 기억해줘" → save_memory 발화·즉시 저장 → 새 대화 "내 생일 언제게?" → 암묵 주입 회상
5. **강건성·폴백·회귀**: 임베딩 서버 다운 상태 회상(LIKE 경로·크래시 0) / 일반 인사에 tool 과호출·기억 노이즈 0 / 기존 채팅·브리핑·웹검색 회귀 0

**결함 검증 항목**(리뷰 추가분): pending 큐 중단 복구(처리 중 강제종료→재시작 시 재처리·중복 저장 0) · LIKE 한국어 2자 부분일치 · save_memory+web_search 라운드 예산 경쟁 · 부정문 추출 정확 · 임베딩 차원 가드 · last_accessed 세션당 1회 · HITL 삭제=물리 DELETE 확인 · 캐시(주입 순서 고정) 유지
