# 리뷰 통합 · 상충판정 가이드

세 리뷰(내부 Claude / agy=Gemini / grok)를 **단일 통합 리뷰**로 합치는 방법. 핵심은 "취합"이 아니라 **사실 판정**이다 — 외부 모델도 틀리거나 과장한다. 통합자(strict-reviewer)는 파일을 직접 열어 옳고 그름을 가린다.

## 1. 지적 정규화
각 리뷰의 지적을 `{심각도, 대상위치, 주장, 근거}`로 분해한다. 같은 사안을 다른 말로 적은 것을 **하나로 묶는다**(중복 제거). 위치(파일·섹션)가 같고 주장이 겹치면 동일 지적.

## 2. 합의 / 상충 / 단독 분류

| 분류 | 정의 | 처리 |
|---|---|---|
| **합의** | 2개 이상 리뷰가 같은 지적 | 신뢰도 ↑. 최상위 반영 후보. 단, 셋 다 같은 환각일 수도 있으니 한 번은 사실 확인. |
| **상충** | 한쪽은 위험/결함, 다른쪽은 무해/반대 | **판정 필수**(아래 3). 삭제 금지, 출처 병기. |
| **단독** | 한 리뷰만 제기 | 근거 강하면 채택, 약하면 "미검증/불확실"로 표기. |

## 3. 상충 판정 절차 (가장 중요)
1. **해당 파일·코드를 직접 연다.** 추측으로 판정하지 않는다.
2. 주장과 실제를 대조한다.
3. 판정을 근거와 함께 한 줄로: `[판정] 채택 / 과장 / 기각 — 근거(파일:위치)`.
4. 통합 리뷰의 "상충·기각 항목"에 **출처(누가 주장)** 와 **판정 근거**를 함께 적는다.

> 실사례(2026-06-05, 이 repo): Gemini "위험1: Anthropic은 프록시 없으면 404"  vs  Grok "과장, 네이티브 지원". → `llama-server --help`에 `--reasoning`/`--jinja` 존재 + `llm-serving/README.md`가 네이티브 `/v1/messages` 명시 확인 → **판정: 과장**. 단 "E2E 실측 게이트는 필요"는 양쪽 다 타당 → 채택.
>
> 실사례: Gemini "위험6: `<think>` 파싱 UI 필수(Phase 7)"  vs  본문 "THINK=off 기본". → 서버 기본값 확인 → **판정: 범위 밖(THINK=on일 때만)**, Phase 10으로 하향.

## 4. 심각도 재조정
외부 리뷰의 심각도 라벨을 그대로 믿지 말고, **영향 기준으로 재평가**한다:
- Blocker = 이대로 가면 빌드 실패·런타임 오류·잘못된 전제로 되돌림.
- Major = 구현 중 반드시 보완.
- Minor = 정합성·가독성.
한 리뷰가 Major라 해도, 판정 결과 영향이 작으면 Minor로 내리고 사유를 적는다(반대도 동일).

## 5. 환각 필터
외부 리뷰가 **존재하지 않는 파일·함수·API·플래그**를 단정하면, 확인 후 기각하고 "환각(검증: …)"으로 적는다. 통합 리뷰에 환각을 그대로 옮기지 않는다.

## 6. 통합 리뷰 출력 템플릿
```markdown
# <대상> 통합 엄중 리뷰
리뷰일: YYYY-MM-DD
입력: 내부(Claude) + 외부(grok, gemini) — 각 원본: plans/<name>-review-{grok,gemini}.md

## 1. 총평 (+ Definition of Ready 판단)
## 2. Blocker (착수 전 필수)
| # | 지적 | 위치 | 근거 | 출처(합의/판정) | 권고 |
## 3. Major
## 4. Minor / 정합성
## 5. 상충 · 기각 항목 (출처 병기 + 판정 근거)
| 쟁점 | A 주장 | B 주장 | 판정 | 근거 |
## 6. 외부 리뷰 원본
- grok: plans/<name>-review-grok.md
- gemini: plans/<name>-review-gemini.md
```

## 7. 통합의 황금률
- **삭제하지 말고 병기**: 상충·소수의견도 출처와 함께 남긴다(나중에 옳았을 수 있다).
- **근거 없는 단정 금지**: 판정은 파일 확인 후에만.
- **과장도 축소도 안 됨**: 위험을 부풀리지도, 실제 위험을 묻지도 않는다.
- **실행 가능하게**: 각 Blocker/Major에 "무엇을 어떻게"가 있어야 한다.
